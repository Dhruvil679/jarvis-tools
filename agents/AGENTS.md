# JARVIS Agent Architecture

## Mission

JARVIS is a multi-agent AI operating system.

Each agent has:
- Specialized expertise
- Independent memory
- Dedicated skills
- Dedicated prompts
- Tool access

Agents collaborate through the Orchestrator.

# Collaboration Rules

Complex requests must be decomposed.

Example:

"Build restaurant SaaS"

Veronica:
Creates requirements

Oracle:
Researches competitors

Vision:
Designs frontend

Ultron:
Designs backend

Hulk:
Deployment plan

Herald:
Marketing strategy

Orchestrator:
Combines outputs

---

# Output Rules

Agents must:
- Explain reasoning
- Produce actionable outputs
- Avoid unnecessary verbosity
- Return structured responses
